## Manual Response Guidelines

When responding manually to user inquiries, follow these guidelines:

### Response Time
- **Target:** Within 48 business hours
- **Priority Issues:** Within 24 hours (data loss, critical bugs)
- **Business Hours:** Monday-Friday, 9 AM - 5 PM PST

### Response Template Structure

```
Hi [User Name],

Thank you for reaching out about [specific issue].

[Provide specific answer or solution]

[Additional relevant information]

If you have any other questions, please don't hesitate to ask.

Best regards,
[Your Name]
Optical Rx Now Support Team
```

### Common Support Scenarios

#### 1. Data Loss Issues
- Emphasize local storage nature
- Explain that data cannot be recovered from servers (none exist)
- Suggest device backup restoration if available
- Recommend regular sharing/emailing of important prescriptions

#### 2. Permission Issues
- Provide step-by-step instructions for granting permissions in device settings
- Reassure users about privacy (permissions only used as stated)
- Clarify which permissions are optional

#### 3. Feature Requests
- Thank user for feedback
- Explain current capabilities
- Note feature request for future consideration
- Do NOT promise specific features or timelines

#### 4. Bug Reports
- Thank user for reporting
- Request detailed information (device, OS, steps to reproduce)
- Provide workaround if available
- Escalate to development team

#### 5. Privacy Concerns
- Reiterate local-only storage
- Reference privacy policy
- Explain zero data collection
- Provide reassurance about data security

### Tone Guidelines
- Friendly and professional
- Patient and understanding
- Clear and concise
- Privacy-focused
- Solution-oriented

### Legal Disclaimers

Always include when relevant:

**Medical Disclaimer:**
"Optical Rx Now is for storing prescription information only and does not provide medical advice. Please consult with a qualified eye care professional for any medical concerns."

**Liability Disclaimer:**
"We are not responsible for data loss resulting from device failure, uninstallation, or other circumstances. Users are responsible for backing up their devices and important information."
