## Overlay Text Guidelines

### Best Practices
- **Font:** Use clean, readable sans-serif fonts (Helvetica, SF Pro, Roboto)
- **Size:** Large enough to read on small thumbnails (minimum 40-60pt)
- **Color:** High contrast with background (white text on dark background or vice versa)
- **Positioning:** Top 20% or bottom 20% of screenshot (avoid middle)
- **Length:** Keep text short (5-7 words maximum)
- **Consistency:** Use same font, color, and style across all screenshots

### Optional Elements
- App icon or logo
- Feature icons or emoji (üì∏, üîí, üë®‚Äçüë©‚Äçüëß‚Äçüë¶)
- Gradient overlays behind text
- Device frames (iPhone/Android bezels)
- Background colors matching app theme

### What to Avoid
- Too much text (cluttered)
- Low contrast (hard to read)
- Covering important UI elements
- Inconsistent styling
- Misleading information

## Overlay Text Guidelines

### Text Best Practices
- **Font Size:** Large and readable (40-80pt for title)
- **Font Weight:** Bold or semi-bold for titles
- **Color:** High contrast with background
- **Background:** Semi-transparent overlay if needed
- **Length:** Keep it short - 3-5 words max
- **Language:** Avoid technical jargon

### Design Principles
- **Consistency:** Use same fonts and colors across all screenshots
- **Branding:** Match app's color scheme
- **White Space:** Don't overcrowd with text
- **Focus:** Text should enhance, not obscure the app
- **Hierarchy:** Title larger than subtitle

### Example Color Scheme
- **Primary:** #0A1628 (Dark blue - matches app)
- **Accent:** #4A90E2 (Light blue)
- **Text:** White on dark, Dark on light
- **Overlay:** rgba(0, 0, 0, 0.3) - semi-transparent black

## Tips for Great Screenshots

### Visual Appeal
1. **Use real UI:** Actual app screenshots look more authentic than mockups
2. **Show functionality:** Demonstrate features in action, not just static screens
3. **Highlight key features:** Use visual cues (arrows, highlights) sparingly
4. **Maintain consistency:** Same data, style, and quality across all screenshots
5. **Tell a story:** Screenshots should flow logically (setup ‚Üí use ‚Üí benefit)

### Technical Quality
1. **Retina/HD quality:** Always use highest resolution available
2. **Proper orientation:** Portrait for phone apps (landscape if app supports it)
3. **Clean backgrounds:** Remove distractions from screenshots
4. **Proper lighting:** Bright, clear, well-lit UI
5. **Color accuracy:** Ensure colors match actual app

### Marketing Impact
1. **Lead with benefit:** First screenshot should show main value proposition
2. **Feature variety:** Show different aspects of app (don't repeat screens)
3. **Privacy emphasis:** At least one screenshot highlighting privacy/security
4. **User perspective:** Show what users will actually see and do
5. **Call to action:** Overlay text should encourage download
