# Screenshot Planning Guide

**For Apple App Store and Google Play Store Submissions**

---

## Screenshot Requirements

### Apple App Store

#### iPhone Screenshots
**Required Sizes (as of 2026):**
- 6.7" Display (iPhone 15 Pro Max, 14 Pro Max): **1290 x 2796 pixels**
- 6.5" Display (iPhone 11 Pro Max, XS Max): 1242 x 2688 pixels
- 5.5" Display (iPhone 8 Plus): 1242 x 2208 pixels (still supported for compatibility)

**Minimum Required:** 2-8 screenshots per device size
**Recommended:** 4-6 screenshots highlighting key features

#### iPad Screenshots (Optional but Recommended)
- 12.9" Display (iPad Pro): **2048 x 2732 pixels**
- 11" Display (iPad Pro): 1668 x 2388 pixels

### Google Play Store

#### Phone Screenshots
**Required Sizes:**
- **Minimum:** 320 x 320 pixels
- **Maximum:** 3840 x 3840 pixels
- **Recommended:** 1080 x 1920 pixels (16:9 ratio) or 1080 x 2340 pixels (19.5:9 ratio)

**Quantity:** 2-8 screenshots minimum

#### Tablet Screenshots (Recommended)
- 7" Tablet: 1200 x 1920 pixels
- 10" Tablet: 1600 x 2560 pixels

**Quantity:** 1-8 screenshots

## Screenshot Creation Tools

### Option 1: Device Screenshots (Recommended)
**Tools:**
- iOS Simulator (Xcode)
- Android Emulator (Android Studio)
- Physical devices (iPhone/Android)

**Steps:**
1. Build app in development mode
2. Populate with sample data
3. Navigate to each screen
4. Take screenshots (Cmd+S in simulators)
5. Transfer to computer

### Option 2: Screenshot Design Tools
**Tools:**
- **Figma** (free, web-based)
- **Sketch** (Mac only, paid)
- **Adobe XD** (free)
- **Screenshot Maker Apps** (App Store, Previewed, etc.)

**Steps:**
1. Design mockups matching app UI
2. Add sample prescription data
3. Export at required sizes
4. Add overlay text and graphics

### Option 3: Screenshot Capture + Editing
**Tools:**
- Device screenshots + Photoshop/GIMP/Canva
- Screenshot frameworks (fastlane snapshot)

**Steps:**
1. Capture raw screenshots from app
2. Edit in design tool
3. Add overlay text, graphics, backgrounds
4. Export at required sizes

## Overlay Text Guidelines

### Best Practices
- **Font:** Use clean, readable sans-serif fonts (Helvetica, SF Pro, Roboto)
- **Size:** Large enough to read on small thumbnails (minimum 40-60pt)
- **Color:** High contrast with background (white text on dark background or vice versa)
- **Positioning:** Top 20% or bottom 20% of screenshot (avoid middle)
- **Length:** Keep text short (5-7 words maximum)
- **Consistency:** Use same font, color, and style across all screenshots

### Optional Elements
- App icon or logo
- Feature icons or emoji (üì∏, üîí, üë®‚Äçüë©‚Äçüëß‚Äçüë¶)
- Gradient overlays behind text
- Device frames (iPhone/Android bezels)
- Background colors matching app theme

### What to Avoid
- Too much text (cluttered)
- Low contrast (hard to read)
- Covering important UI elements
- Inconsistent styling
- Misleading information

## Screenshot Checklist

### Before Taking Screenshots
- [ ] App populated with realistic sample data
- [ ] All features working correctly
- [ ] UI is clean and free of debug information
- [ ] Device in light mode (recommended for screenshots)
- [ ] Status bar clean (full battery, strong signal, appropriate time)
- [ ] Keyboard hidden unless demonstrating input
- [ ] No personal or sensitive information visible

### Screenshot Quality
- [ ] High resolution (meeting platform requirements)
- [ ] No pixelation or blurriness
- [ ] Proper aspect ratio
- [ ] Correct device frame (if using frames)
- [ ] Consistent style across all screenshots

### Content Review
- [ ] No spelling or grammatical errors
- [ ] Accurate representation of app features
- [ ] No placeholder or "Lorem ipsum" text
- [ ] Professional and polished appearance
- [ ] Compliant with app store guidelines

### Platform Compliance
- [ ] No inappropriate content
- [ ] No competing platform references (no "Android" in iOS screenshots)
- [ ] No misleading claims
- [ ] Medical disclaimer visible if showing medical data
- [ ] Age-appropriate content

## Tips for Great Screenshots

### Visual Appeal
1. **Use real UI:** Actual app screenshots look more authentic than mockups
2. **Show functionality:** Demonstrate features in action, not just static screens
3. **Highlight key features:** Use visual cues (arrows, highlights) sparingly
4. **Maintain consistency:** Same data, style, and quality across all screenshots
5. **Tell a story:** Screenshots should flow logically (setup ‚Üí use ‚Üí benefit)

### Technical Quality
1. **Retina/HD quality:** Always use highest resolution available
2. **Proper orientation:** Portrait for phone apps (landscape if app supports it)
3. **Clean backgrounds:** Remove distractions from screenshots
4. **Proper lighting:** Bright, clear, well-lit UI
5. **Color accuracy:** Ensure colors match actual app

### Marketing Impact
1. **Lead with benefit:** First screenshot should show main value proposition
2. **Feature variety:** Show different aspects of app (don't repeat screens)
3. **Privacy emphasis:** At least one screenshot highlighting privacy/security
4. **User perspective:** Show what users will actually see and do
5. **Call to action:** Overlay text should encourage download

## Screenshot Testing

Before submission, verify:
- [ ] View screenshots on App Store/Play Store preview tool

- [ ] Verify text is readable at small sizes
- [ ] Test on different device screen sizes
- [ ] Get feedback from others (clarity, appeal, accuracy)
- [ ] Ensure compliance with store guidelines
