## Required Screenshot Sizes

### iOS App Store

#### iPhone
- **6.7" Display (iPhone 15 Pro Max, 14 Pro Max, etc.)**
  - Size: 1290 x 2796 pixels
  - Required: 3-10 screenshots

- **6.5" Display (iPhone 11 Pro Max, XS Max, etc.)**
  - Size: 1284 x 2778 pixels
  - Optional but recommended

- **5.5" Display (iPhone 8 Plus, 7 Plus, etc.)**
  - Size: 1242 x 2208 pixels
  - Optional for older device support

#### iPad (Optional but Recommended)
- **12.9" Display (iPad Pro)**
  - Size: 2048 x 2732 pixels
  - Required if supporting iPad: 3-10 screenshots

- **11" Display (iPad Pro, iPad Air)**
  - Size: 1668 x 2388 pixels
  - Optional

### Google Play Store

#### Phone
- **Minimum:** 320px
- **Maximum:** 3840px
- **Recommended:** 1080 x 1920 pixels or 1080 x 2400 pixels
- **Required:** 2-8 screenshots

#### Tablet (Optional)
- **Recommended:** 1600 x 2560 pixels or 2048 x 2732 pixels
- **Required if supporting tablets:** 2-8 screenshots

## Tools for Creating Screenshots

### Capture Screenshots

#### iOS
1. **Xcode Simulator**
   - Run app in simulator
   - `Device ‚Üí Screenshot` or `Cmd + S`
   - Automatically correct size

2. **Physical Device**
   - Run app on iPhone
   - Press Volume Up + Side Button
   - Transfer via AirDrop or USB

#### Android
1. **Android Emulator**
   - Run app in Android Studio emulator
   - Camera icon in emulator toolbar
   - Saves to Screenshots folder

2. **Physical Device**
   - Run app on Android phone
   - Press Volume Down + Power Button
   - Transfer via USB or cloud storage

### Add Overlay Text & Polish

#### Professional Tools
1. **Figma** (Free) - https://figma.com
   - Import screenshots
   - Add text overlays
   - Design templates available
   - Export in required sizes

2. **Canva** (Free/Paid) - https://canva.com
   - Mobile app mockup templates
   - Easy text overlay
   - Professional fonts
   - Brand colors

3. **Sketch** (macOS, Paid)
   - Professional design tool
   - App screenshot templates
   - Precise control

4. **Adobe Photoshop** (Paid)
   - Advanced editing
   - Full control over design

#### Quick & Easy Tools
1. **Preview** (macOS)
   - Basic text and shapes
   - Quick annotations

2. **Apple Keynote** (Free on macOS/iOS)
   - Slide templates
   - Easy text overlay
   - Export as images

## Overlay Text Guidelines

### Text Best Practices
- **Font Size:** Large and readable (40-80pt for title)
- **Font Weight:** Bold or semi-bold for titles
- **Color:** High contrast with background
- **Background:** Semi-transparent overlay if needed
- **Length:** Keep it short - 3-5 words max
- **Language:** Avoid technical jargon

### Design Principles
- **Consistency:** Use same fonts and colors across all screenshots
- **Branding:** Match app's color scheme
- **White Space:** Don't overcrowd with text
- **Focus:** Text should enhance, not obscure the app
- **Hierarchy:** Title larger than subtitle

### Example Color Scheme
- **Primary:** #0A1628 (Dark blue - matches app)
- **Accent:** #4A90E2 (Light blue)
- **Text:** White on dark, Dark on light
- **Overlay:** rgba(0, 0, 0, 0.3) - semi-transparent black

## Tips for Great Screenshots

### Do's ‚úÖ
- Use real-looking sample data (but anonymized)
- Show the app in actual use
- Highlight key features
- Use high-quality images
- Test on different display sizes
- Show diverse family members
- Keep UI clean and uncluttered
- Use professional fonts
- Maintain consistent branding
- Export at exact required dimensions

### Don'ts ‚ùå
- Don't use real personal data
- Don't show error states
- Don't include outdated UI
- Don't use pixelated images
- Don't over-compress files
- Don't use too much text
- Don't obscure important UI elements
- Don't use inconsistent styling
- Don't forget to localize if supporting multiple languages
